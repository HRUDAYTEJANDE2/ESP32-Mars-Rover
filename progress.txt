- Added ESP-NOW communication between base and rover
- The base ESP sends integers based on the command it receives:-
1 = left, 2 = right...
The integers are sent to the rover, which executes the required function

Command -> Base -> (string to integer) -> Rover (integer to command)

- All the commands are now inside void output(int input) which executes the command for the given integer input

PROBLEMS
- After I made the code, I was confused as to why my code outputted LEFT instead of the given command. I soon realized that I used '=' instead of '==' *sighs*
- My rover's ESP kept crashing, but I figured it out now.

The ESP32 crashed (kernel panic) when executing the rleft and rright commands. The serial monitor showed a watchdog reset and the program stopped responding to new ESP-NOW commands.

The code used while(true) loops inside the rleft and rright functions, blocking the main() function.

I removed the rleft and rright codes from output(), and added a variable where the rover mode is stored (mode) instead of calling main.a . The main loop() continuously checks this state and runs the required output.
